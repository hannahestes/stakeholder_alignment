# 3D Pareto Front Analysis: Accuracy × Simplicity × Stability

## Executive Summary

This analysis explores the **trade-off landscape** of decision trees generated by Explainable Rule Extraction (EZR) on the BankChurners dataset. By running EZR 50 times, we discover that multiple competing solutions exist, each with different strengths:

- Some trees are **highly accurate** but complex and unstable
- Some trees are **simple and stable** but less accurate  
- Some trees achieve an **optimal balance** across all three dimensions

This naturally supports **"Convergent Divergence"**: teams can converge on uncertainty (multiple valid solutions exist) while diverging on how to prioritize accuracy vs. simplicity vs. reliability.

## Methodology: Feature Frequency Stability

Unlike traditional feature-set matching (which penalizes natural variation), we use **feature frequency stability**:

```
For each tree, stability = average frequency of its features

Example:
Tree uses [income_Category, education_Level, marital_Status]
- income_Category appears in 90% of runs
- education_Level appears in 72% of runs  
- marital_Status appears in 48% of runs

Stability = (90 + 72 + 48) / 3 = 70%

Interpretation: This tree uses reliable core features.
```

**Why this works**:
- ✅ Rewards trees that use frequently-selected features
- ✅ Penalizes trees that rely on rare features
- ✅ Captures genuine consensus vs. exploratory variation
- ✅ More realistic than exact feature-set matching

## Technical Details

### Pareto Dominance (3D)

Tree B **dominates** Tree A if:
- B has **≥ accuracy** AND
- B has **≤ features** (simpler) AND  
- B has **≥ stability** AND
- **at least one is strictly better**

**Frontier trees**: No other tree dominates them.

### Feature Frequency Calculation

For each feature across 50 runs:
```
frequency = (count of runs where feature appears) / 50 * 100
```

Example:
- `income_Category` appears in 45/50 runs → 90% frequency
- `marital_Status` appears in 24/50 runs → 48% frequency

### Stability Calculation

For each tree:
```
stability = average frequency of its features

Tree using [income (90%), education (72%), marital (48%)]
stability = (90 + 72 + 48) / 3 = 70%
```
